1. Traversal
    We traverse the token list left → right, ingesting tokens one by one:
        - We create a new t_cmd when needed
        - Each t_cmd contains:
        - argv (WORDS)
        - redirections (list)
        - WORD → added to argv
        - REDIRECTION → consumes next WORD as filename, appended to redirections
        - PIPE → ends current command and starts a new one
        - Syntax errors are handled during traversal
2. Components
    A. Pipes
        Rules:
            Pipe splits commands
            Pipe must be surrounded by valid commands
        Errors:
            Pipe at start
            Pipe at end
            Consecutive pipes (||)
            Pipe when no valid command exists on left
            Pipe followed by invalid start (e.g. nothing)
    B. Redirections
        Rules:
            Redirection must be followed by WORD
            WORD is consumed as filename only
            Parsing continues normally after that
            Multiple redirections allowed
            Redirections are applied in order (last overrides in effect)
        Errors:
            Redirection not followed by WORD
    C. Words
        Rules:
            WORD goes into argv unless consumed by redirection
            First WORD defines command
            Remaining WORDs are arguments
        Errors:
            none at syntax level
3. Command definition (implicit from your design)
    A command is:  everything between pipes, built incrementally during traversal
    Contains:
        1. argv
        2. ordered redirections
